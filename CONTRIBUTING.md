# Contributing to Open Weald Software

Thank you for your interest in contributing to projects maintained by Open Weald Software.

This organization develops open source tools intended to support civic infrastructure, community institutions, and projects that serve the public good. Our primary project is Commons Atlas.

We welcome contributions that help improve the reliability, clarity, and long-term usefulness of the software.

---

## Before You Begin

Before submitting a contribution, please take a moment to review the existing documentation and open issues.

Early-stage projects benefit from coordination. Opening an issue or discussion before beginning significant work helps ensure that proposed changes align with the direction of the project.

---

## Ways to Contribute

Contributions are not limited to code. Useful contributions may include:

- Reporting bugs  
- Proposing new features or improvements  
- Improving documentation  
- Clarifying workflows or domain concepts  
- Identifying usability issues  
- Providing feedback from real-world use cases  

Practitioners working in civic infrastructure, public administration, nonprofit operations, or community development are particularly encouraged to share feedback.

---

## Reporting Issues

Please use the provided issue templates when reporting bugs or requesting features.

Structured templates help ensure that maintainers and contributors have the information necessary to understand and reproduce problems.

Issues that lack sufficient detail to reproduce the problem may be closed. Reports such as “the app does not work” or vague interface complaints without context do not provide enough information to investigate.

---

## Submitting Pull Requests

Pull requests are welcome for bug fixes, documentation improvements, and approved feature work.

Before submitting a pull request:

- Ensure the change addresses a documented issue or discussion  
- Keep changes focused and well-scoped  
- Include a clear description of the changes being made  

If you intend to submit a substantial pull request, please open an issue or discussion before beginning work.

Major changes typically include:

- Architectural modifications  
- Database schema changes  
- Large refactors  
- New features affecting multiple parts of the system  

Early discussion helps ensure proposed work aligns with the direction of the project and prevents contributors from spending time on changes that may not be accepted.

---

## Design Philosophy

Projects maintained by Open Weald Software follow several guiding principles:

- Durability over novelty  
- Clear and explicit system models  
- Software designed for real institutional work  
- Long-term maintainability  

Contributions that align with these principles are more likely to be accepted.

---

## Code of Conduct

All participation in this project is governed by the organization's Code of Conduct.

Please review the Code of Conduct before participating.

---

## Licensing

By contributing to a repository in this organization, you agree that your contributions will be licensed under the project's open source license as defined in the repository.

---

## Questions

If you are unsure how to contribute, opening an issue or discussion to ask questions is always welcome.

Thoughtful discussion often leads to the most useful improvements.
